데이터베이스 생성을 누릅니다.

이제 Django에서 RDS에 접근해 봅시다.

settings.py에 들어가 DATABASES 코드를 수정합니다.

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'db명',
        'USER': '마스터 사용자 이름',
        'PASSWORD': '마스터 암호',
        'HOST': '엔드포인트',
        'PORT': '3306',
        'OPTIONS': {
            'init_command': 'SET sql_mode="STRICT_TRANS_TABLES"'
        }
    }
}

엔드포인트는 생성된 데이터베이스 인스턴스를 누르면 확인할 수 있습니다.

이제 migrate를 적용해 변경사항을 업데이트 해줍시다.

python3 manage.py migrate

에러 관련

  1. Error loading MySQLdb module

위 에러가 난다면 mysql 모듈이 설치되지 않아 생기는 오류입니다.

아래 명령어를 이용해 mysql 모듈을 설치해줍니다.

pip3 install Mysqlclient

  1. pymysql.err.OperationalError: (2003,"Can't connect to MySQL server on~~~

이런 에러가 난다면 생성한 db가 외부에서 접속이 안되기 때문에 생기는 에러입니다.

생성한 db인스턴스의 보안 그룹을 수정해줍시다.

이것과 같이 규칙을 추가하고 저장해줍시다.

  1. django.db.utils.InternalError: (1049, "Unknown database

이 에러는 db를 찾지 못해 생기는 에러입니다. db 인스턴스는 생성했지만 스키마가 존재하지 않아 접근할 db를 찾지 못하는 것입니다.

쉬운 작업을 위해 mysql workbench를 설치합니다.

MySQL Connections 옆에 + 버튼을 눌러 연결해줍시다.

Connection Name: 아무 이름이나

Hostname: 생성한 db 인스턴스의 엔드포인트를 넣어줍니다.

Username: db인스턴스를 생성할 때 만든 마스터 사용자 이름

Password: db인스턴스를 생성할 때 만든 마스터 비밀번호

OK를 누르고 들어간 후

db 생성 버튼을 누릅니다.

Schema Name: 새롭게 생성할 스키마 이름을 적어줍니다.

스키마 이름은 위에서 Django setting.py에서 DATABASE에 ‘Name’에 넣어주면 됩니다.
