배포할 Django 프로젝트 만들기

  • 가상환경 폴더 만들기
$ python3 -m venv myvenv
  • myvenv 자리엔 원하는 가상환경 명을 입력하시면 됩니다.
  • 가상환경 활성화 방법
  • mac의 경우
$ source myvenv/bin/activate
  • windows의 경우
$ source myvenv/Scripts/activate

터미널에서 아래와 같이 표시된다면 가상환경이 활성화 된 겁니다.

(myvenv) [현재 위치]$

장고 설치

$ pip3 install django

배포할 프로젝트 생성 프로젝트 명은 원하는 것을 입력하시면 됩니다.

$ django-admin startproject mysite

프로젝트에 앱(app)을 만듭니다.

$ cd mysite
$ python3 manage.py startapp app

settings.py 에서 app 앱을 추가하고 사용하는 언어와 timezone을 수정합니다. 사이트가 영어만을 사용한다면 LANGUAGE_CODE는 변경하지 않아도 됩니다.

배포 성공시 체크할 간단한 템플릿을 만들겠습니다.

app 폴더 안에 templates 폴더를 만들고, 그 안에 main 폴더를 만든 뒤 index.html을 아래와 같이 만드세요.

app/templates/main/index.html

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <h1>~배포 성공~</h1>
</body>
</html>

views와 urls설정을 합니다.

app/views.py

from django.contrib import admin
from django.urls import path
from main import views as main_views

urlpatterns = [
    path('', main_views.index, name="index"),
    path('admin/', admin.site.urls),
]

migrate 해준 뒤, runserver를 해줍니다.

$ python3 manage.py makemigrations
$ python3 manage.py migrate
$ python3 manage.py runserver

프로젝트를 업로드할 준비가 되면, 프로젝트에 사용한 패키지와 라이브러리들을 서버 컴퓨터에도 똑같이 설치해야 하기 때문에 설치한 패키지와 라이브러리들 목록을 저장하기 위해 manage.py가 있는 폴더에서

$ pip3 freeze >> requirements.txt

이 명령어를 입력하면 현재 활성화된 가상환경에 설치된 패키지들을 requirements.txt에 저장할 수 있습니다.

AWS 서버 대여하기

aws에 로그인하고, EC2를 검색해 들어갑니다.

서비스 리전을 서울로 선택해줍니다.

인스턴스를 클릭해 들어갑니다. 인스턴스 = 컴퓨터라고 생각하시면 됩니다.

인스턴스 시작을 클릭합니다.

더 많은 AMI 찾아보기를 클릭하고

Ubuntu server라고 입력하고

제일 상단에 있는 “Ubuntu Server 22.04 LTS (HVM), SSD Volume Type”를 선택해주세요.

이제 키 페어를 생성해야 합니다. 새 키 페어 생성을 클릭하고,

키 페어 이름을 입력하고 키 페어 생성 버튼을 누르면 키 페어가 다운 받아집니다.

이 키 페어는 여러분이 임대한 AWS EC2 서버에 원격으로 접근하기 위한 키입니다. 절대 잃어버리면 안되고, 외부에 공개해서도 안됩니다.

이제 나머지 설정들은 건들지 마시고 인스턴스 시작 버튼을 누르면 성공적으로 인스턴스가 실행됩니다.

이제 다운받은 키 페어를 관리해봅시다.

다운로드 폴더에 있는 키 페어을 홈 폴더의 .ssh 폴더로 옮깁니다. 만약 홈 폴더에 .ssh 폴더가 없다면 생성한 뒤 옮겨줍니다.

(~/.ssh 폴더가 없을 경우) $ mkdir ~/.ssh/
$ mv ~/Downloads/test.pem ~/.ssh/

키 페어 파일의 권한을 소유주만 읽을 수 있도록 변경합니다.

$ chmod 400 ~/.ssh/test.pem

AWS EC2 서버에 원격 접속하기

터미널에서 다음 명령어를 입력하면 원격으로 접속 할 수 있습니다.

ssh -i [키 페어 경로] [유저 이름]@[퍼플릭 DNS 주소]

초기 유저 이름은 ubuntu로 설정되어 있습니다.

따라서 저의 경우는 .ssh폴더에 test.pem이 있으니 다음 명령어와 같습니다

ssh -i ~/.ssh/test.pem ubuntu@ec2-43-201-24-115.ap-northeast-2.compute.amazonaws.com

uWSGI 서버 연결하기

Django는 웹 서버와 직접적으로 통신할 수 없기에 중간에 WSGI server를 설치해 주어야 합니다.

$ source ~/myvenv/bin/activate
$ pip3 install uwsgi

uwsgi 서버를 이용해 django 서버를 키려면 다음 형식의 명령어를 치시면 됩니다.

$ uwsgi --http :[포트번호] --home [가상환경 경로] --chdir [장고프로젝트폴더 경로] -w [wsgi 모듈이 있는 폴더].wsgi

즉, 그대로 적용하면 저의 명령어는 아래와 같습니다.

$ uwsgi --http :8080 --home /home/ubuntu/myvenv/ --chdir /srv/Back/ -w mysite.wsgi
