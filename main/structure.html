📌  MAIN (LOGIN) 구조 설명 : 코드 작성 단계


1. 패키지 import

1) 코드의 시작 부분에서 필요한 패키지들을 import 합니다.

2) 필요하다면 pubspec.yaml에 패키지들을 추가한 후, 터미널에서 flutter pub get 명령어를 사용하여 프로젝트의 의존성을 최신 버전으로 업그레이드합니다.


2. main 함수

1) 앱을 실행하고, ‘MyApp’ 클래스를 호출하여 앱의 기본 구조를 정의하는 메인 함수를 작성합니다.


3. MyApp 클래스

1) 앱의 전반적인 테마와 홈 화면을 설정합니다.

2) 로그인 한 사용자만 해당 앱의 기능을 이용할 수 있도록 ‘MyLogin’ 클래스의 인스턴스를 홈 화면으로 설정합니다.


4. MyLogin 클래스

1) 초기 라우트로 로그인 페이지(’LoginPage’)를 설정합니다.


5. LoginPage 클래스

1) StatefulWidget을 상속받아 로그인 페이지를 구현합니다.

2) ‘TextEditingController’를 활용하여 사용자로부터 이메일과 비밀번호를 입력받습니다.


6. _LoginPageState 클래스

1) ‘LoginPage’의 상태를 관리하는 클래스를 선언합니다.


7. showResultDialog 함수 (로그인 시도 결과 표시 함수)

1) 로그인 결과를 사용자에게 팝업으로 알려주기 위한 함수를 작성합니다.

2) 서버로부터 성공 또는 실패 메시지를 전달받아 AlertDialog를 생성합니다.

3) 로그인 성공 시 홈 화면으로 이동하도록 설정합니다.

4) ‘확인’ 버튼을 포함하여 버튼 클릭 시 팝업이 닫히도록 설정합니다.


8. login 함수 (로그인 시도 함수)

1) 서버에 로그인 요청을 보내는 비동기 함수를 작성합니다.

2) HTTP POST를 사용하여 서버에 이메일과 비밀번호 정보를 전송하고, 응답에 따라 성공 또는 실패 메시지를 출력합니다.

3) 사용자 랭킹 정보를 랭킹 페이지에서 불러오기 위해 로그인 성공 시 사용자 이메일을 SharedPreferences를 사용하여 저장합니다.


9. build 함수

1) 화면의 UI를 정의하는 함수를 작성합니다.

2) Scaffold를 사용하여 앱의 기본 구조를 정의합니다.

3) Scaffold 위에 AppBar와 이메일, 비밀번호 입력필드, 회원가입 및 로그인 버튼을 배치합니다.

4) 회원가입 버튼을 누르면 ‘SignupPage’로 이동하고, 로그인 버튼을 누르면 ‘login’ 함수가 호출되도록 설정합니다.


10. Widget 구성

1) TextField : 사용자로부터 이메일과 비밀번호를 입력받는 데 사용되는 위젯입니다.

2) ElevatedButton : 회원가입 및 로그인 버튼을 나타내는 위젯으로, 회원가입 페이지로 이동하거나 로그인 시도가 가능하도록 구현합니다.
